# Deployment Guide\n\n## Vercel Deployment\n\n### Prerequisites\n\n- Vercel account\n- GitHub repository connected to Vercel\n- PostgreSQL database (Vercel Postgres, AWS RDS, or other provider)\n\n### Steps\n\n1. **Connect Repository to Vercel**\n   - Go to https://vercel.com/new\n   - Import the GitHub repository\n   - Select the project\n\n2. **Configure Environment Variables**\n   - Set `DATABASE_URL` to your PostgreSQL connection string\n   - Set `JWT_SECRET` to a secure random string (min 32 characters)\n   - Set `NODE_ENV` to `production`\n   - Set `NEXT_PUBLIC_API_URL` to your production domain\n\n3. **Deploy**\n   - Click \"Deploy\"\n   - Vercel will automatically build and deploy\n\n4. **Run Migrations**\n   - After deployment, run migrations on your database:\n   ```bash\n   npm run db:generate\n   npm run db:migrate\n   ```\n\n## Docker Deployment\n\n### Dockerfile\n\n```dockerfile\nFROM node:18-alpine\n\nWORKDIR /app\n\nCOPY package*.json ./\nRUN npm ci\n\nCOPY . .\nRUN npm run build\n\nEXPOSE 3000\n\nCMD [\"npm\", \"start\"]\n```\n\n### Build and Run\n\n```bash\ndocker build -t wifi-voucher-platform .\ndocker run -p 3000:3000 -e DATABASE_URL=\"...\" -e JWT_SECRET=\"...\" wifi-voucher-platform\n```\n\n## Traditional Server Deployment\n\n### Prerequisites\n\n- Node.js 18+ installed\n- PostgreSQL database\n- SSL certificate (recommended)\n\n### Steps\n\n1. **Clone Repository**\n   ```bash\n   git clone https://github.com/kelgralich1122/Wifi-voucher-.git\n   cd Wifi-voucher-\n   ```\n\n2. **Install Dependencies**\n   ```bash\n   npm install\n   ```\n\n3. **Set Environment Variables**\n   ```bash\n   export DATABASE_URL=\"postgresql://...\"\n   export JWT_SECRET=\"your-secret-key\"\n   export NODE_ENV=\"production\"\n   ```\n\n4. **Build Application**\n   ```bash\n   npm run build\n   ```\n\n5. **Run Migrations**\n   ```bash\n   npm run db:generate\n   npm run db:migrate\n   ```\n\n6. **Start Server**\n   ```bash\n   npm start\n   ```\n\n## Production Checklist\n\n- [ ] Set strong `JWT_SECRET` (min 32 chars, random)\n- [ ] Enable HTTPS/SSL\n- [ ] Configure database backups\n- [ ] Set up monitoring/logging\n- [ ] Configure rate limiting\n- [ ] Set environment to `production`\n- [ ] Use environment variables for all secrets\n- [ ] Enable CORS if needed\n- [ ] Set up CI/CD pipeline\n- [ ] Configure error tracking (Sentry, etc.)\n\n## Database Backup\n\n### PostgreSQL Backup\n\n```bash\npg_dump -U postgres -h localhost -d app_db > backup.sql\n```\n\n### PostgreSQL Restore\n\n```bash\npsql -U postgres -h localhost -d app_db < backup.sql\n```\n\n## Monitoring\n\n### Health Check\n\nThe application provides a health check endpoint:\n```bash\ncurl https://your-domain.com/api/health\n```\n\n## Scaling\n\nFor high-traffic deployments:\n\n1. Use a load balancer\n2. Run multiple instances\n3. Use connection pooling (PgBouncer)\n4. Implement caching (Redis)\n5. Use a CDN for static assets\n\n## Security\n\n- Always use HTTPS in production\n- Keep dependencies updated\n- Use strong passwords for database\n- Implement rate limiting\n- Enable audit logging\n- Regular security audits\n- Keep backups encrypted\n
